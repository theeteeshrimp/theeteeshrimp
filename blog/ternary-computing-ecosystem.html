1. Introduction

For over 80 years, binary logic has been the bedrock of computing. Every processor, every programming language, every operating system in mainstream use today operates on the same principle: two states, 0 and 1, on and off. But this dominance is more a product of historical engineering convenience than mathematical necessity.

Ternary computing — computation based on three logical states — offers a fundamentally different approach. Instead of bits, ternary computers use trits, each capable of representing three values. The most elegant form, balanced ternary, uses the digits −1, 0, and +1 (often written as T, 0, 1). Donald Knuth himself called balanced ternary "perhaps the prettiest number system" in The Art of Computer Programming.

Despite its theoretical advantages, ternary computing has remained largely in the shadows — a fascinating "what if" of computer science. But a growing ecosystem of emulators, programming languages, compilers, and even operating systems has been built by researchers, hobbyists, and companies exploring what a post-binary world might look like.

This report surveys that ecosystem in full: from the historical Setun computers of the Soviet Union to modern CNTFET-based ternary logic gates, from educational Python simulators to ambitious full-stack software platforms.

2. Foundations: Ternary Logic & Number Systems

2.1 Unbalanced vs. Balanced Ternary

There are two primary flavours of ternary representation:

SystemDigitsNotes
Unbalanced ternary0, 1, 2Direct base-3 extension of binary
Balanced ternary−1, 0, +1Symmetric around zero; no sign bit needed
Fractional unbalanced0, ½, 1Specialised for certain DSP applications

Balanced ternary is the more interesting system for general-purpose computing. Its key advantages:

  • No separate sign bit: Negative numbers are represented by inverting all trits. The number −5 in balanced ternary is simply the inversion of +5.
  • Rounding becomes truncation: In balanced ternary, truncating the least-significant trit is equivalent to rounding to the nearest value — a property that simplifies floating-point arithmetic.
  • Symmetric arithmetic: Addition and subtraction use the same hardware, since subtraction is just addition of the inverted trit string.

2.2 Information Density

One trit encodes log₂(3) ≈ 1.585 bits of information. This means:

  • 3 trits = 27 values (vs. 3 bits = 8 values)
  • 6 trits = 729 values (vs. 6 bits = 64 values)
  • 9 trits (a "tryte") = 19,683 values (vs. 9 bits = 512 values)

A 27-trit word can represent over 7.6 trillion values — equivalent to ~42 binary bits. This higher information density means ternary systems can represent the same range of values with fewer digits, potentially reducing the width of data paths and the complexity of arithmetic circuits.

2.3 Ternary Logic Gates

Just as binary logic has AND, OR, and NOT gates, ternary logic has its own set of fundamental gates. In balanced ternary:

  • NOT (inversion): Maps −1→+1, 0→0, +1→−1
  • AND (min): Returns the minimum of two inputs
  • OR (max): Returns the maximum of two inputs
  • CYCLE: Rotates the input through the three states
  • COMPARE: Tests equality of two trits

These gates form the basis of all ternary computation, from simple adders to full ALUs.

3. Historical Context: The Setun Legacy

3.1 Setun (1958)

The Setun (Сетунь) was the world's first modern ternary computer, developed at Moscow State University in 1958 under the leadership of Sergei Sobolev and Nikolay Brusentsov. It was built using ferrite-diode magnetic elements and operated on balanced ternary logic.

Key specifications:

  • Word size: 18 trits (6 trytes)
  • Memory: 81 words of RAM (ferrite core)
  • Speed: 2,000 operations per second
  • Power consumption: Just 75 watts (remarkably low for the era)

The Setun was not a lab curiosity — it was used for real scientific computing at Moscow State University for over a decade. Fifty units were produced, and they proved remarkably reliable and cost-effective compared to their binary contemporaries.

3.2 Setun-70 (1970)

The Setun-70 was a more advanced successor, introducing a structured programming architecture based on a two-stack model. It featured:

  • DSSP (Dialog System for Structured Programming): An interactive programming environment that allowed structured, conversational programming — remarkably ahead of its time.
  • Nastavnik ("Teacher"): A computer-assisted teaching system designed to help students learn programming on the Setun-70.
  • Stack-based architecture: The Setun-70 used a two-stack design (data stack and return stack) that anticipated later stack-based computers and Forth-like systems.

The Setun-70's programming system was arguably more sophisticated than what most binary computers of the era offered. DSSP provided an interactive, structured programming experience that wouldn't become mainstream until decades later.

3.3 Why Setun Was Suppressed

Despite its technical success, the Setun project was eventually sidelined. The Soviet computing establishment favoured binary systems for compatibility with the broader Eastern Bloc computing infrastructure. The last Setun was decommissioned in 1970, and the project was largely forgotten in the West until the internet age.

4. Emulators & Simulators

Since actual ternary hardware is extremely rare, emulators and simulators are the primary way most people interact with ternary computing. A rich ecosystem has developed over the past two decades.

4.1 Tunguska

Tunguska is the most comprehensive ternary computing emulator suite, originally developed around 2008. It is loosely inspired by the 6502 processor design but fully ternary. The name is a nod to the predominantly Russian heritage of ternary computing.

ComponentDescription
EmulatorFull ternary virtual machine with memory management
AssemblerCompiles assembly code into compressed binary format
3CCExperimental C-like cross-compiler for the Tunguska architecture
OSA crude operating system written in assembly, providing basic I/O routines

Tunguska's architecture uses a 6-trit word size (tryte), giving it a 3-trit address space. The emulator provides a complete programming environment: write assembly, assemble it, and run it on the virtual machine — all on a standard binary computer.

4.2 Triador (ssloy)

Triador is a ternary computer emulator with a 3-trit architecture, designed to aid in debugging hardware designs. It's notable for its simplicity and educational value: 3-trit registers (27 values each), step-by-step execution showing complete state, and example programs.

4.3 Trinity (JohnSully)

Trinity is a ternary computer project on GitHub that explores ternary CPU design and simulation. It focuses on the processor architecture level, simulating ternary logic gates and their composition into functional units like ALUs and registers.

4.4 brandonssmith/ternary

A simple ternary computer simulator written in Python, implementing balanced ternary logic (−1, 0, +1) for educational purposes. One of the most accessible entry points for programmers who want to understand ternary computing without diving into complex emulator suites.

4.5 tedkotz/ternary

Ted Kotz's ternary experiments repository contains an emulator for a 27-trit balanced ternary computer. It includes a library for performing ternary operations and serves as a testbed for ternary algorithms, exploring larger word sizes than most educational emulators.

4.6 Tern Systems / BTMC

The Tern Computer organisation on GitHub hosts the BTMC (Balanced Ternary Machine Code) project, which provides specifications, tools, and examples for a modern ternary computing architecture. One of the more ambitious attempts to create a complete, well-documented ternary computing specification.

4.7 Louis Duret-Robert's Ternary Projects

Louis Duret-Robert has built a comprehensive suite of ternary computing tools, including:

  • TernaryVerilog: HDL-level ternary processor design and circuit optimisation
  • Ternary ALU: Working ternary arithmetic logic unit design
  • Ternary logic circuit designer and simulator: Python/Tkinter application for designing and simulating ternary circuits
  • PCB designs for ternary gates: Physical circuit board layouts for ternary logic gates

This work bridges the gap between software emulation and actual hardware implementation.

5. Programming Languages

5.1 Owlet (nirvanasupermind)

Owlet is a programming language specifically designed for ternary computing. Created by nirvanasupermind and published on npm, Owlet targets the balanced ternary number system and provides native balanced ternary arithmetic, ternary-native data structures, and a compiler that targets ternary machine code. One of the first serious attempts to create a high-level language natively designed for ternary architectures.

5.2 Bruijn

Bruijn is a purely functional programming language based on lambda calculus and de Bruijn indices, written in Haskell. While not exclusively a ternary language, it uses balanced ternary as its native number representation and compiles to Tromp's binary lambda calculus. Features include: pure lambda calculus foundation with no built-in functions, balanced ternary as the native number system, extremely small core reducer, and architecture-independent design that could run on ternary hardware.

5.3 Ternary (Esolang)

Ternary is an esoteric programming language created by zerosum0x0 in 2015. It's a member of the TrivialBrainfuckSubstitution family, where Brainfuck commands are encoded using only the characters 0, 1, and 2. While not intended for practical use, it demonstrates how ternary encoding can represent any computation.

5.4 DSSP (Dialog System for Structured Programming)

DSSP was the programming system for the Setun-70 computer. It was an interactive, structured programming environment that allowed programmers to write code in a conversational manner, supporting structured programming constructs (loops, conditionals, procedures) and interactive development. Remarkably modern in its approach — interactive structured programming environments didn't become mainstream in the West until the 1980s.

5.5 3CC (Tunguska C Compiler)

3CC is an experimental C-like cross-compiler for the Tunguska ternary architecture. It allows programmers to write C-like code that compiles to Tunguska machine code, providing a higher-level alternative to assembly programming for the emulator.

6. Compilers & Toolchains

6.1 MVL Synthesis Tool (USN)

The University of South Norway (USN) Ternary Research Group has developed an open-source MVL (Multiple-Valued Logic) synthesis tool that generates binary, ternary, and hybrid (mixed radix) circuits using Carbon Nanotube FETs (CNTFETs). It takes ternary logic function specifications as input and outputs SPICE-compatible netlist files for simulation.

6.2 uMemristorToolbox

Also from the USN Ternary Research Group, uMemristorToolbox is an open-source framework for reading and writing non-volatile ternary states to memristors. This is significant because memristors are one of the most promising physical implementations of ternary logic — they can naturally store three (or more) distinct resistance states.

6.3 TernaryVerilog

Louis Duret-Robert's TernaryVerilog is a circuit optimiser for ternary logic designs expressed in Verilog HDL. It takes ternary logic descriptions and optimises them for area, power, and delay — analogous to how binary logic synthesis tools work in conventional EDA flows.

7. Operating Systems & System Software

7.1 Tunguska OS

The Tunguska emulator includes a basic operating system written in ternary assembly. While "crude" by modern standards, it provides basic I/O routines (character input/output), memory management functions, and a slightly less hostile environment than bare memory. It demonstrates the fundamental principles of ternary system software.

7.2 Nastavnik ("Teacher")

The Nastavnik system on the Setun-70 was a computer-assisted teaching system. While not a general-purpose OS, it provided interactive programming lessons, a program editing and execution environment, and student progress tracking. One of the earliest examples of computer-assisted instruction, predating similar systems on binary computers by years.

7.3 Conceptual Ternary OS Design

No full-featured ternary operating system exists today comparable to Linux or Windows. However, the research community has outlined what such a system would require:

  • Ternary memory management: Page tables and virtual memory designed for trit-addressable memory
  • Ternary file systems: Data structures optimised for ternary storage (tryte-aligned blocks rather than byte-aligned)
  • Ternary instruction set architecture: A complete ISA with ternary opcodes, addressing modes, and privilege levels
  • Ternary device drivers: I/O interfaces designed for three-state signalling

The fundamental challenge is that every layer of the software stack — from the bootloader to the filesystem to the application runtime — would need to be designed from scratch for ternary architectures.

8. Modern Research & Hardware

8.1 CNTFET-Based Ternary Logic

Carbon Nanotube Field-Effect Transistors (CNTFETs) are one of the most promising technologies for implementing ternary logic in hardware. Research has shown that ternary logic gates using CNTFETs can achieve 2000× less delay and 104× less power compared to existing designs. CNTFETs can be fabricated using modified versions of existing CMOS processes, and hybrid binary/ternary circuits are possible on the same chip.

8.2 Memristor-Based Ternary Memory

Memristors — resistors with memory — can naturally store multiple resistance states, making them ideal for ternary (and higher-base) logic. Recent work includes ternary memory controllers for memristors in hardware, hybrid memristor-CNTFET ternary logic gates, and non-volatile ternary state storage with uMemristorToolbox.

8.3 Samsung/UNIST Ternary Semiconductor

In 2019, a research team led by Kim Kyung-rok at Ulsan National Institute of Science and Technology (UNIST) in South Korea, with Samsung backing, demonstrated the world's first unbalanced ternary semiconductor design on a large wafer. This was a landmark achievement, showing that ternary logic could be manufactured using modified versions of existing chipmaking processes.

8.4 Ternary Computing for AI

Ternary computing has found a surprising new application in AI and machine learning:

  • Ternary neural networks: Replacing floating-point weights with {−1, 0, +1} values can dramatically reduce model size and computation cost
  • Microsoft's BitNet: Explores extreme quantisation of large language models, with ternary being a natural target
  • Ternary logic aligns with neural network weight representation: The three states map naturally to "inhibit", "neutral", and "excite" in neural models

8.5 China's Non-Binary AI Chip (2025)

In 2025, a team led by Professor Li Hongge at Beihang University in Beijing initiated mass production of what was reported as the world's first non-binary AI chip, integrating hybrid computing technology into critical sectors including aviation and industrial systems.

8.6 Huawei's 7nm Ternary Chips

Huawei has reportedly been developing ternary chips at the 7nm process node, demonstrating a viable path toward commercial ternary hardware.

8.7 Ternary Optical Computer (TOC)

The Ternary Optical Computer uses optical fibre rings as registers, semiconductor memory as ternary cells, and liquid crystal as modulator and adder. This approach leverages the natural three-state properties of certain optical systems to perform ternary computation.

9. The Ternary Software Stack: Tidal & Tern Systems

9.1 Tern Computer

Tern Computer (tern.ac) is a company building what they call "the world's first ternary software stack." Their Tidal platform aims to provide a complete software stack designed for ternary hardware, development tools for ternary-native applications, and a bridge between existing binary infrastructure and future ternary hardware.

9.2 BTMC (Balanced Ternary Machine Code)

The BTMC project from Tern Systems provides specifications and tools for a balanced ternary machine code architecture, including instruction set specification, assembly language tools, and example programs and documentation.

9.3 USN Ternary Research Group

The University of South Norway's Ternary Research Group has been actively researching ternary computing and Multiple-Valued Logic (MVL) for over a decade. Their contributions include open-source MVL logic synthesis tools, memristor-based ternary memory frameworks, ternary robotics control systems, and automated netlist synthesis for ternary circuits.

10. Challenges & Outlook

10.1 The Noise Problem

The fundamental challenge for ternary computing is noise margin. With binary logic, the gap between a 0 and a 1 is large relative to typical noise levels. With ternary logic, the three states must be closer together, making them more susceptible to noise-induced errors. This is why binary won out historically — it's simply more robust in the presence of electrical noise.

10.2 The Infrastructure Problem

Eight decades of binary computing have created an enormous ecosystem of tools, languages, compilers, operating systems, and applications. Rebuilding all of this for ternary computing is a monumental task. Every piece of the stack — from logic gates to programming languages to operating systems — needs to be redesigned.

10.3 The Compatibility Problem

Any ternary computer needs to interoperate with the existing binary world. This means ternary systems need binary interfaces, file format converters, and network protocol implementations — all adding complexity.

10.4 The Path Forward

Despite these challenges, several factors are driving renewed interest in ternary computing:

  1. Moore's Law slowdown: As binary transistors approach physical limits, alternative computing paradigms are increasingly attractive
  2. AI workloads: Ternary neural networks offer dramatic efficiency improvements for AI inference
  3. New hardware technologies: CNTFETs, memristors, and optical computing make ternary logic more practical than ever
  4. Energy efficiency: Ternary systems can potentially do more computation per unit of energy

The most likely path to mainstream ternary computing is through hybrid systems — chips that combine binary and ternary logic, using ternary for specific workloads (like AI inference) where it offers clear advantages.
